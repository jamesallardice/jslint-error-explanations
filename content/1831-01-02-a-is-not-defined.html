When do I get this error?

JSHint will throw the "'{a}' is not defined" error when it encounters a reference to an identifier that has not been declared as part of a var or function statement, and the undef option is set to true.

This is the equivalent of the JSLint "'{a}' was used before it was defined" warning. More detail can be found on the page for that message.

In JSHint 1.0.0 and above you have the ability to ignore any warning with a special option syntax. The identifier of this warning is W117. This means you can tell JSHint to not issue this warning with the /*jshint -W117 */ directive.
