Why do I get this error?

This error is raised to highlight a fatal syntax error. The language specification shows the grammar for regular expression literals (ES5 §7.8.5):

RegularExpressionLiteral ::
    / RegularExpressionBody / RegularExpressionFlags

This production makes it clear that regular expression literals must be terminated by a / character. Not doing so will always cause a syntax error. To fix this issue, simply close the regular expression in question:

In JSHint 1.0.0 and above you have the ability to ignore any warning with a special option syntax. Since this message relates to a fatal syntax error you cannot disable it.
